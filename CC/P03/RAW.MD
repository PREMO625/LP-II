## Theory

### What Is Apex?

Apex is a proprietary language developed by Salesforce.com. It is a strongly typed, object-oriented programming language that allows developers to execute flow and transaction control statements on the Force.com platform server in conjunction with calls to the Force.com API.

It has a Java-like syntax and acts like database stored procedures. It enables developers to add business logic to system events such as button clicks, related record updates, and Visualforce pages. Apex code can be initiated by web service requests and from triggers on objects. Apex is included in Performance Edition, Unlimited Edition, Enterprise Edition, and Developer Edition.

### Features of Apex

#### Java-like Syntax and Easy to Use

Uses Java-style variable declarations, loops, and conditionals.

#### Strongly Integrated With Data

Apex is data-focused and designed to execute multiple queries and DML statements together.

#### Strongly Typed

Direct references to schema objects are validated, and invalid references fail quickly.

#### Multitenant Environment

The runtime engine prevents runaway code from monopolizing shared resources and enforces limits.

#### Automatic Upgrades

Apex is upgraded as part of Salesforce releases with no manual upgrade needed.

#### Easy Testing

Apex provides built-in unit testing support, code coverage reports, and guidance on efficiency.

### When Should a Developer Choose Apex?

Apex code should be written when the business scenario is too complex for built-in Salesforce functionality.

Common scenarios:

- Create web services to integrate Salesforce with other applications.
- Implement custom validation on sObjects.
- Execute custom Apex logic during DML operations.
- Implement features not possible with workflows, flows, or process builders.
- Process email contents, headers, and attachments using Apex email services.

### Flow of Actions for Apex Code

**Developer Action**

When a developer saves Apex code, it is compiled into instructions understood by the Apex runtime interpreter. These instructions are saved as metadata on the platform.

**End User Action**

When a user event triggers Apex code, the platform server retrieves the compiled instructions, runs them through the Apex interpreter, and returns the result.
