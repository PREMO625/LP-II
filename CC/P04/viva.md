# Oral Viva Guide - CC P04 (Salesforce Mini Project)

## Quick One-Liners

- **Salesforce**: Cloud CRM platform.
- **Lightning**: Modern Salesforce UI.
- **Custom Object**: User-defined data table.
- **Field**: Column in object.
- **Record**: Row in object.
- **Tab/App**: UI access to objects.

---

## Basic Questions

1. What is Salesforce?
   - A cloud CRM platform for sales, service, and apps.
2. What is the Lightning Platform?
   - Salesforce PaaS for building custom apps.
3. What is a custom object?
   - A user-defined object to store data.
4. What is a field?
   - An attribute/column of an object.
5. What is a record?
   - A row of data in an object.
6. What is a tab?
   - UI entry point to access an object.
7. What is a Lightning App?
   - A collection of tabs and items in the Lightning UI.
8. What is the Object Manager?
   - Setup area to manage objects and fields.
9. What is the role of Setup?
   - Configuration and administration console.
10. Why use Developer Edition?
    - Free environment for practice and learning.

---

## Practical Flow Questions

11. Why did we create a custom object?
    - To store student records in a structured way.
12. Why are fields created after the object?
    - Fields define the data structure of the object.
13. Why create a custom tab?
    - To make the object accessible in the UI.
14. Why add the tab to the Sales app?
    - So users can access it in the Lightning App.
15. Why create sample records?
    - To demonstrate data entry and output.
16. What is the record name field?
    - The primary identifier (Student Name).
17. Why is Marks a Number field?
    - Marks are numeric values.
18. Why is Grade a Text field?
    - Grades are string values.
19. What happens if you skip the tab?
    - The object exists but is harder to access in UI.
20. What is the output of this practical?
    - The Student Records list view screen.

---

## Intermediate Questions

21. What is Salesforce Classic vs Lightning?
    - Classic is older UI; Lightning is modern UI.
22. What is a profile in Salesforce?
    - Set of permissions for a user.
23. Why assign apps to profiles?
    - To control who can see and use the app.
24. What is App Manager?
    - Manage Lightning apps and their settings.
25. What is a list view?
    - A table view of records.
26. Can you add validation rules?
    - Yes, for data quality control.
27. What is a page layout?
    - The form layout for record creation/view.
28. What is a field data type?
    - Defines how data is stored (text, number, date).
29. What is a schema in Salesforce?
    - The structure of objects, fields, and relationships.
30. Why use declarative tools?
    - Faster setup without coding.

---

## Advanced and External Questions

31. What is a master-detail relationship?
    - A parent-child relationship with shared ownership.
32. What is a lookup relationship?
    - A loose link between objects.
33. What is a validation rule?
    - Logic that prevents incorrect data entry.
34. What is a workflow rule?
    - Automates actions based on criteria.
35. Can Lightning apps include standard objects?
    - Yes.
36. What is a report type?
    - Defines which objects can be reported together.
37. How do you control field visibility?
    - Field-level security and page layouts.
38. What is a sandbox?
    - A testing environment separate from production.
39. When do you use Apex in such apps?
    - When declarative tools are not enough.
40. What is a governor limit?
    - Platform resource limits to protect multi-tenancy.
